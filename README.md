# WhatsApp-ссылка бот

Телеграм-бот, который по номеру телефона и типу заведения (кафе / салон
красоты) генерирует готовую ссылку на WhatsApp с заранее написанным текстом
для холодного предложения сайта.

## Как пользоваться (со стороны бота)

1. Пишете боту что угодно (например `/start`) — если это первый раз, он
   спросит ваше имя. Это имя будет автоматически подставляться в текст
   сообщения вместо "Михаил".
2. Отправляете номер телефона клиента в любом виде:
   `+7 701 204 50 64`, `87012045064`, `7012045064` — бот сам уберёт пробелы,
   скобки, тире и плюс, и приведёт к виду `77012045064`.
3. Бот спрашивает: **Кафе** или **Салон красоты** (кнопки).
4. После выбора бот присылает готовую ссылку вида:
   `https://api.whatsapp.com/send/?phone=77012045064&text=...&type=phone_number&app_absent=0`
   — нажав на неё, вы откроете WhatsApp с уже вписанным текстом.
5. Чтобы поменять сохранённое имя — отправьте `/name`.

## Запуск бота

1. Получите токен у [@BotFather](https://t.me/BotFather) в Telegram
   (команда `/newbot`).
2. Установите зависимости:
   ```
   pip install -r requirements.txt
   ```
3. Укажите токен и запустите бота:
   ```
   export BOT_TOKEN="ваш_токен_от_BotFather"
   python bot.py
   ```
   (на Windows вместо `export` используйте `set BOT_TOKEN=...`)
4. Бот заработает в режиме polling — оставьте процесс запущенным
   (например, на сервере через `screen`, `tmux` или systemd-сервис).

## Деплой на Railway

Бот работает в режиме polling (сам стучится в Telegram), поэтому ему **не
нужен публичный порт/домен** — Railway просто держит процесс запущенным.

1. Залейте эти файлы в GitHub-репозиторий (включая `Procfile`,
   `requirements.txt`, `runtime.txt`).
2. На [railway.app](https://railway.app) → **New Project** → **Deploy from
   GitHub repo** → выберите свой репозиторий.
3. Railway сам распознает Python-проект (через Nixpacks) и найдёт
   `Procfile` — команда запуска `python bot.py` подхватится автоматически.
   Если нет — зайдите в **Settings → Deploy → Start Command** и впишите
   `python bot.py` вручную.
4. Перейдите в **Variables** и добавьте переменную:
   - `BOT_TOKEN` = ваш токен от @BotFather
5. Нажмите **Deploy**. В **Deployments → View Logs** должна появиться строка
   `Бот запущен` — значит всё работает, можно идти проверять в Telegram.
6. (Опционально) В **Settings** отключите генерацию публичного домена —
   она не нужна, бот ничего не принимает по HTTP.

### Важно про хранение имени (`users.json`)

Файловая система Railway сбрасывается при каждом новом деплое
(redeploy/push). Это значит, что после обновления кода файл `users.json`
исчезнет, и бот заново спросит у вас имя при следующем сообщении — для
одного пользователя это не страшно.

Если хотите, чтобы имя сохранялось навсегда даже после редеплоев — добавьте
в Railway **Volume** (Settings → Volumes → New Volume, путь монтирования
`/data`) и поменяйте в `bot.py` путь:
```python
USERS_FILE = "/data/users.json"
```
Так файл будет жить на отдельном диске, который не пересоздаётся при
деплое.

## Где хранятся имена пользователей

Имена сохраняются в файле `users.json` рядом со скриптом — формат
`{"telegram_id": "Имя"}`. Можно открыть и поправить вручную при необходимости.

## Как изменить тексты сообщений

Тексты для кафе и салона красоты находятся в начале `bot.py`, в словаре
`TEMPLATES`. Место для имени помечено как `{name}` — не удаляйте эту часть,
иначе подстановка имени перестанет работать.
